# Palantir Dashboard — Project Memory

## What This Is
A static React dashboard tracking Palantir government & defense contracts globally.
No build system — React 18, Recharts, Babel all loaded from CDN.

## File Structure
```
palantir-dashboard/
  index.html          — entry point, loads all scripts
  app.jsx             — entire React app (~78KB), single component
  data.js             — window.CONTRACTS (50+ contracts), window.COLORS, window.RUN_RATES
  palantir-docs.js    — window.PALANTIR_OFFICIAL_DOCS (earnings, SEC filings, letters)
  pending.js          — window.PENDING_ITEMS (auto-generated by scraper daily)
  sources_config.json — all scraper sources (X accounts, RSS feeds, contract APIs)
  start.bat           — double-click to launch dashboard (also has desktop shortcut)

scraper/
  main.py             — orchestrator, runs all scrapers, writes pending.js
  seen_ids.json       — tracks already-processed item IDs (auto-generated)
  requirements.txt
  scrapers/
    us_spending.py        — USASpending.gov API
    uk_contracts.py       — UK Contracts Finder API
    eu_ted.py             — EU TED API
    canada.py             — CanadaBuys / Open Canada
    australia.py          — AusTender
    ukraine_prozorro.py   — Prozorro
    sec_edgar.py          — Palantir 8-K filings (CIK: 0001321655)
    palantir_ir.py        — investors.palantir.com press releases
    rss_feeds.py          — Substack + Medium + news RSS
    web_search.py         — DuckDuckGo → X posts + defense media

.github/workflows/daily_scrape.yml — GitHub Actions, runs at 8am UTC daily
```

## Dashboard Tabs (in order)
1. Overview — stat cards + 4 charts
2. Explorer — filterable/sortable contract table
3. By Country — contracts grouped by country
4. Timeline — vertical timeline of all contracts
5. Deal Flow — charts: contracts/year, status, procurement type, product frequency
6. Run Rate — quarterly run rate analysis
7. PLTR Docs — earnings reports, shareholder letters, SEC filings, IR press releases
8. Sources — source references
9. **Inbox** — NEW: daily scraper review queue (approve/decline items)

## sources_config_data.js
- New file: loaded by index.html as window.SOURCES_CONFIG
- Rich metadata for every source: descriptions, platform, focus area, palantir_specific flag, filter flag, frequency, coverage
- 27 X accounts, 16 RSS/newsletter feeds, 6 contract APIs, 2 official disclosure sources

## Inbox Tab
- Reads from window.PENDING_ITEMS (pending.js)
- Approve/Decline state stored in localStorage (keys: pltr_approved, pltr_declined)
- Pink count badge on tab when pending items exist
- Filters: Pending / Approved / Declined / All + source type filter
- Each item: source badge, country, value, title, snippet, Open Source link, Approve/Decline buttons
- Export Approved button: always visible, active when approved contract_api items exist → modal with JS code to paste into data.js
- Daily Sources panel: below inbox items, shows all 51 configured sources with descriptions, sort by Type/Name/Category/Inbox Items, filter by category; shows live contribution counts from pendingItems

## Scraper Architecture
- Tier 1 (contract APIs): USASpending, UK Contracts Finder, EU TED, CanadaBuys, AusTender, Prozorro
- Tier 2 (official disclosures): SEC EDGAR 8-K, Palantir IR press releases
- Tier 3 (media): RSS feeds (Substack/Medium/news), DuckDuckGo web/X search
- Output: pending.js with max 500 items, rolling window
- Deduplication via seen_ids.json (committed to repo)

## How to Run
- Dashboard: double-click start.bat (or desktop shortcut) → http://localhost:8000
- Scraper (manual): `pip install -r scraper/requirements.txt` then `python scraper/main.py`
- Scraper (auto): push to GitHub → GitHub Actions runs daily at 8am UTC
