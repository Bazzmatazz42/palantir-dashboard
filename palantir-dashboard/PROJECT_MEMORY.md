# Palantir Dashboard — Project Memory

## What This Is
A static React dashboard tracking Palantir government & defense contracts globally.
No build system — React 18, Recharts, Babel all loaded from CDN.

## File Structure
```
palantir-dashboard/
  index.html              — entry point, loads all scripts
  app.jsx                 — entire React app (~97KB), single component
  data.js                 — window.CONTRACTS (462 contracts), window.COLORS, window.RUN_RATES
  palantir-docs.js        — window.PALANTIR_OFFICIAL_DOCS (earnings, SEC filings, letters)
  pending.js              — window.PENDING_ITEMS (auto-generated by scraper daily)
  sources_config.json     — all scraper sources (X accounts, RSS feeds, podcasts, contract APIs)
  sources_config_data.js  — window.SOURCES_CONFIG with rich metadata for dashboard Sources panel
  start.bat               — double-click to launch dashboard locally → http://localhost:8000

scraper/
  main.py             — orchestrator, runs all scrapers, writes pending.js
  seen_ids.json       — tracks already-processed item IDs (gitignored, auto-generated)
  requirements.txt
  scrapers/
    us_spending.py        — USASpending.gov API (full history, cursor pagination, up to 20 pages)
    uk_contracts.py       — UK Contracts Finder API
    eu_ted.py             — EU TED API
    canada.py             — CanadaBuys / Open Canada
    australia.py          — AusTender
    ukraine_prozorro.py   — Prozorro
    sec_edgar.py          — Palantir 8-K filings (CIK: 0001321655)
    palantir_ir.py        — investors.palantir.com press releases
    rss_feeds.py          — Substack + Medium + news RSS
    web_search.py         — DuckDuckGo → X posts + defense media

.github/workflows/daily_scrape.yml — GitHub Actions, runs at 8am UTC daily
index.html (repo root)             — redirects to palantir-dashboard/ for GitHub Pages
.nojekyll (repo root)              — disables Jekyll on GitHub Pages
```

## Source Registry
- sources_master.json: 68 sources, single source of truth
  - 7 inbox (contract APIs, SEC EDGAR)
  - 26 screened (defense media RSS, leadership, tier-1 X handles → screener routes)
  - 35 karptube (analyst newsletters, Reddit, YouTube, DDG queries, tier-2 X handles)
- sources_master.js: window.SOURCES_MASTER wrapper, auto-regenerated by scraper
- source_registry.py: load(), get_rss_sources(), get_youtube_sources(), get_reddit_sources(),
  get_x_handles(), get_ddg_queries(), get_contract_apis(), get_disclosure_sources(),
  update_stats(), save() — tracks 90-day per-source history
- main.py + karptube_scraper.py: both read from source_registry, no hardcoded lists
- Daily scraper commits sources_master.json + sources_master.js with updated stats

## Scraper Architecture
- Tier 1 (contract APIs): USASpending (working), UK/EU/Canada/Australia/Prozorro (APIs broken/auth required — fall back to DDG)
- Tier 2 (official disclosures): SEC EDGAR 8-K, Palantir IR press releases
- Tier 3 (media): RSS feeds (requests pre-fetch → feedparser), DDG news queries (8 broad + 2 X-specific)
- Output: pending.js max 500, karptube.js max 1000, rolling windows
- Deduplication via seen_ids.json + karptube_seen_ids.json (persists in repo via Actions commit)
- Typical yield: ~50 items/run from Google News RSS (35) + DDG news (60 raw → deduped ~15)

## Inbox Tab
- Reads from window.PENDING_ITEMS (pending.js)
- Approve/Decline state stored in localStorage (keys: pltr_approved, pltr_declined)
- Pink count badge on tab when pending items exist
- Filters: Pending / Approved / Declined / All + source type filter
- Export Approved button: always visible, generates JS to paste into data.js
- Daily Sources panel: all configured sources with descriptions, sort/filter, live contribution counts

## KarpTube Tab
- Tab exists in app.jsx, reads from window.KARPTUBE_ITEMS (karptube.js)
- karptube_scraper.py runs independently via GitHub Actions, writes to karptube.js
- Wide-net: Google News, defense RSS, Substacks, Reddit, YouTube, DuckDuckGo
- Rolling 1000-item window, 7-day cutoff

## Screener (scraper/screener.py)
- Added deal/revenue signal screener to main.py pipeline
- score >= 3 → Inbox (pending.js); score < 3 → KarpTube (karptube.js)
- Auto-pass: contract_api, sec_edgar source types (always Inbox)
- Palantir IR: auto-pass only if deal terms present
- Scored signals: contract language (+3), dollar values (+3), deal actions (+2),
  customer names (+2), expansion language (+1), business metrics (+1)
- Noise penalty: stock price commentary, retail investor talk, etc.
- Customer list: all US cabinet depts (incl. Dept of War — renamed from DoD 2025),
  all military branches/combatant commands, full IC list, law enforcement,
  civilian agencies, international allies, prime contractors
- screen_score and screen_reason fields stamped on every item
